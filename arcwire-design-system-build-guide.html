Claude Code — Setup

Storybook Component Setup

How to use Claude Code to scaffold component templates that mirror the Figma boolean/variant logic.

How this mirrors Figma

Every boolean prop in Figma becomes a boolean prop in code. Every variant becomes a string union type. Every mode becomes a CSS variable theme. The Figma logic IS the component API.

Initial Claude Code Commands
✓
1. Init Storybook in repo
Run in Claude Code terminal to scaffold Storybook with the right config for a Shopify/Liquid + Web Components approach.
Claude Code — terminal
# Run in your project root npx storybook@latest init # Select: HTML / Web Components / Vite # Then install token tooling: npm install style-dictionary @tokens-studio/sd-transforms
✓
2. Generate token files from Figma variables
Export from Figma using Tokens Studio plugin → JSON → run Style Dictionary to generate CSS custom properties.
Claude Code prompt
"Create a Style Dictionary config that: - Reads tokens/figma-export.json - Outputs CSS variables for each brand mode (minimal, standard, premium, editorial, wellness) - Outputs a tokens.ts type file - Nests output as: tokens/dist/[mode].css"

Token Architecture in Code

How the Figma variable modes map to CSS architecture.

CSS Token Structure (mirrors Figma variable modes)
/* tokens/dist/base.css — primitives */ :root { --color-blue-500: #3b82f6; --color-neutral-50: #fafafa; --radius-sm: 4px; --radius-md: 8px; --space-4: 16px; } /* tokens/dist/minimal.css — brand mode */ [data-theme="minimal"] { --bg-primary: #ffffff; --text-primary: #0a0a0a; --radius-component: 0px; --font-heading: 'Neue Haas Grotesk', sans-serif; } /* tokens/dist/wellness.css — brand mode */ [data-theme="wellness"] { --bg-primary: #f7f3ef; --text-primary: #2d2420; --radius-component: 16px; --font-heading: 'Canela', serif; }

Component Templates

PDP Buy Section — props map to Figma booleans 1:1

Your Figma Info Panel booleans (Breadcrumbs, Eye Brow, Wishlist, Swatches, Size, Subscribe, BNPL, USPS, Description) become the component props below.

Claude Code prompt — PDP Buy Section component
"Create a PDPBuySection web component with these props mirroring the Figma boolean system: Boolean props (all default true): - showBreadcrumbs, showEyebrow, showWishlist - showSwatches, showSize, showSubscribe - showBNLP, showUSPS, showDescription - showReviews, showUrgency, showCrossSell Variant prop (string union): - layout: 'media-left' | 'media-right' | 'editorial' | 'sticky' Mode prop (maps to CSS theme): - theme: 'minimal' | 'standard' | 'premium' | 'editorial' | 'wellness' Output: - pdp-buy-section.js (web component) - PDPBuySection.stories.js (Storybook story) - pdp-buy-section.css (consuming CSS vars) The Storybook story should show all boolean controls in the Storybook controls panel."
Generated component structure (Claude Code output)
class PDPBuySection extends HTMLElement { static get observedAttributes() { return [ 'show-breadcrumbs', 'show-eyebrow', 'show-wishlist', 'show-swatches', 'show-size', 'show-subscribe', 'show-bnpl', 'show-usps', 'show-description', 'show-reviews', 'show-urgency', 'layout', 'theme' ]; } render() { const show = (attr) => this.getAttribute(attr) !== 'false'; this.innerHTML = ` <div class="pdp-buy" data-layout="${this.getAttribute('layout') || 'media-left'}" data-theme="${this.getAttribute('theme') || 'standard'}"> ${show('show-breadcrumbs') ? '<pdp-breadcrumbs/>' : ''} ${show('show-eyebrow') ? '<p class="eyebrow">...</p>' : ''} ${show('show-wishlist') ? '<wishlist-button/>' : ''} ${show('show-swatches') ? '<colour-swatches/>' : ''} ${show('show-size') ? '<size-selector/>' : ''} ${show('show-subscribe') ? '<subscribe-section/>' : ''} ${show('show-bnpl') ? '<bnpl-options/>' : ''} </div> `; } } customElements.define('pdp-buy-section', PDPBuySection);
